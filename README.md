# 📚 도서 관리 시스템 (Bookstore Project) 리팩토링 보고서
본 프로젝트는 기존 레거시 시스템의 치명적인 보안 취약점을 해결하고, 데이터 무결성 및 사용자 경험(UX)을 개선하기 위해 전면적인 리팩토링을 거친 풀스택 웹 애플리케이션입니다.

## 🛠️ 실행 방법 (Getting Started)
프로젝트를 로컬 환경에서 실행하려면 아래 과정을 순서대로 진행해 주세요.

### 1. 필수 라이브러리 설치 및 DB 초기화
```bash
npm install
node src/utils/init-db.js
```
### 2. 환경변수 설정
프로젝트 루트 폴더에 .env 파일을 생성하고 아래 항목들을 설정해 주세요.

```
NEXT_PUBLIC_API_URL=http://localhost:3000/api
ADMIN_TOKEN=your_secret_admin_token_123
NEXT_PUBLIC_ADMIN_ID=your_admin_id_here
NEXT_PUBLIC_ADMIN_PASSWORD=your_admin_password_here
```
### 3. 개발 서버 실행
```bash
npm run dev
```
### 4. 브라우저 접속
서버가 구동되면 http://localhost:3000으로 접속하여 결과를 확인합니다.

📁 프로젝트 구조 (Project Structure)
```Plaintext
src/
├── app/
│   ├── page.tsx          # 메인 UI (도서 목록, 검색, 장바구니, 삭제 기능)
│   ├── globals.css       # 전역 스타일링
│   └── api/
│       ├── books/route.ts # 도서 추가/조회 (백엔드 로직 정상화 완료)

    ├── db.js             # SQLite 연동 및 공통 쿼리 유틸

```
## ✨ 상세 리팩토링 내역 (Improvement Report)

## 1. 🛡️ 보안의 '금고' 설치 (환경변수 및 아키텍처)
> **비유**: 집 열쇠를 현관문 앞에 붙여두던 방식에서, 보이지 않는 안전한 금고에 보관하는 방식으로 변경했습니다.

* **문제**: API 주소가 '내 컴퓨터'로 고정되어 외부 배포가 불가능했고, 비밀번호가 소스 코드에 그대로 노출되어 있었습니다.
* **조치**: 모든 민감 정보(비밀번호, API 주소)를 `.env`라는 전용 금고 파일로 분리했습니다.
* **결과**: 이제 코드 수정 없이 설정값만 바꿔주는 것으로 운영 환경을 즉시 전환할 수 있으며, 해킹으로부터 안전해졌습니다.
* **향후 계획**: 서비스 확장을 위해 AWS 및 외부 전문 데이터베이스(MySQL 등)로의 마이그레이션 기반을 마련했습니다.

---

## 2. 🔑 이중 잠금 장치 도입 (인증 및 보안 강화)
> **비유**: "나 관리자야"라고 말만 하면 들여보내 주던 입구에, 실제 신분증(ID/PW)을 대조하는 보안 요원을 배치했습니다.

* **문제**: 주소창에 특정 글자만 넣으면 관리자 권한을 훔칠 수 있었고, 데이터베이스를 직접 공격하는 'SQL 인젝션' 위협에 노출되어 있었습니다.
* **조치**: 주소창 조작 방식을 폐기하고 **아이디와 비밀번호가 모두 일치**해야만 로그인이 가능하도록 이중 인증을 도입했습니다. 또한, 모든 DB 명령어를 안전하게 보호 처리했습니다.
* **결과**: 비정상적인 권한 탈취가 불가능해졌으며, 상단 바에 로그인 상태를 명확히 표시하여 관리 편의성을 높였습니다.
* **향후 계획**: 비밀번호 암호화(bcrypt) 및 정식 사용자 테이블 구축을 통해 보안 등급을 상향할 예정입니다.

---

## 📝 3. 데이터의 '질' 개선 및 청소 도구 마련 (무결성 및 Reset UX)
> **비유**: 점원이 가격을 대충 짐작해서 부르던 가게에 정확한 스캐너를 도입하고, 오염된 데이터를 한 번에 청소할 '대청소 버튼'을 만들었습니다.

* **문제**: 가격에 글자가 섞여 들어오거나, 서버가 랜덤하게 재고를 만들어내는 오류가 있었습니다. 또한 재고가 '0'일 때 정보가 사라지는 버그가 존재했습니다.
* **조치**: 숫자 전용 입력창을 적용하고, 재고 '0'도 정확히 표시되게 수정했습니다. 특히, 관리자 화면에 **[데이터베이스 초기화]** 기능을 신설하여 보안 토큰 입력 시에만 안전하게 데이터를 청소할 수 있게 했습니다.
* **결과**: 데이터 흐름의 일관성을 확보했으며, 운영 중 발생하는 비정상 데이터를 관리자가 즉각 정리할 수 있는 도구를 확보했습니다.
* **향후 계획**: 데이터베이스 수준에서 제약 조건을 추가하여 잘못된 값이 저장될 가능성을 원천 차단할 예정입니다.

## ⚙️ 4. 시스템의 '기초 체력' 강화 (TypeScript 안정화)
> **비유**: 계기판 없는 자동차를 운전하다가, 모든 부품 상태를 실시간 모니터링하는 최신식 대시보드를 장착했습니다.

* **문제**: 코드를 실행해보기 전까지는 에러를 알 수 없어 시스템이 갑자기 멈추는 일이 잦았습니다.
* **조치**: 모든 데이터에 '이름표(Type)'를 붙여, 개발 단계에서 미리 에러를 발견할 수 있도록 체계를 잡았습니다.
* **결과**: 예상치 못한 시스템 다운(Runtime Error)을 획기적으로 줄여 서비스의 연속성을 확보했습니다.
* **향후 계획**: 임시 이름표를 정교한 모델로 교체하여 결점 없는 시스템으로 고도화할 예정입니다.

## 🛒 5. 스마트한 장바구니 관리 (UX 및 기능 개선)
> **비유**: 담기만 하면 뺄 수 없던 바구니에 '선택 삭제' 버튼을 추가하고, 금액을 읽기 편하게 쉼표(,)를 찍어드렸습니다.

* **문제**: 장바구니 항목 삭제 기능이 없었고, 금액이 `15000`처럼 숫자만 붙어 있어 가독성이 떨어졌습니다.
* **조치**: 개별 상품 옆에 '삭제' 버튼을 추가하고, 모든 금액에 **천 단위 쉼표(예: 15,000원)**를 적용했습니다. 전체 삭제는 관리자만 가능하도록 보안을 강화했습니다.
* **결과**: 사용자의 쇼핑 경험이 쾌적해졌으며, 무분별한 데이터 삭제 위험을 방어했습니다.
* **향후 계획**: 새로고침 후에도 목록이 유지되도록 서버 연동 기능을 강화하고 전역 상태 관리를 도입할 예정입니다.
