# 📚 도서 관리 시스템 리팩토링 및 보안 강화 프로젝트

본 프로젝트는 기존 시스템의 치명적인 보안 취약점을 해결하고, 데이터 무결성 및 사용자 경험(UX)을 개선하기 위해 진행되었습니다.

## 🚀 주요 개선 사항

### 1. 보안 및 아키텍처 (Security & Architecture)
- **환경변수(.env) 도입**: 하드코딩된 관리자 비밀번호 및 API 주소를 분리하여 보안성 향상.
- **SQL Injection 방어**: 파라미터 바인딩 방식을 적용하여 데이터베이스 해킹 위협 차단.
- **인가 우회 차단**: URL 조작을 통한 관리자 권한 탈취 문제를 해결하고 세션 기반 검증 도입.

### 2. 비즈니스 로직 및 데이터 무결성
- **재고 관리 정상화**: 시스템에서 임의로 생성하던 난수 로직을 제거하고, 실제 입력값이 DB에 반영되도록 수정.
- **유효성 검사 강화**: 가격/재고 입력 시 숫자 강제 및 필수값 검증 로직 추가.
- **UI 버그 수정**: 재고가 '0'일 때 정보가 누락되던 현상을 널 병합 연산자(??)로 해결.

### 3. UX/UI 및 타입 안정성
- **장바구니 삭제 기능**: 사용자가 담은 상품을 개별적으로 취소할 수 있는 버튼 및 로직 추가.
- **TypeScript 고도화**: 에러 객체 및 매개변수에 명확한 타입을 정의하여 런타임 안정성 확보.

1. **라이브러리 설치**
   ```bash
   npm install

루트 폴더에 .env 파일을 생성하고 아래 항목을 설정하세요.

## 프로젝트 구조

```
src/
├── app/
│   ├── page.tsx              # 메인 페이지
│   ├── globals.css           # 전역 스타일
│   └── api/
│       ├── books/route.ts    # 도서 API
│       ├── cart/route.ts     # 장바구니 API
│       └── admin/route.ts    # 관리자 API
└── utils/
    ├── db.js                 # DB 유틸 (SQLite 연동)
    └── init-db.js            # DB 초기화 스크립트
bookstore.db                  # SQLite 데이터베이스 파일
```
