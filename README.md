📚 도서 관리 시스템 (Bookstore Project) 리팩토링 보고서

본 프로젝트는 기존 레거시 시스템의 치명적인 보안 취약점을 해결하고, 데이터 무결성 및 사용자 경험(UX)을 개선하기 위해 전면적인 리팩토링을 거친 풀스택 웹 애플리케이션입니다.

🛠️ 실행 방법 (Getting Started)

프로젝트를 로컬 환경에서 실행하려면 아래 과정을 순서대로 진행해 주세요.

1. 필수 라이브러리 설치

npm install


2. 환경변수 설정

프로젝트 루트 폴더에 .env 파일을 생성하고 아래 항목들을 설정해 주세요. (보안을 위해 실제 값은 외부에 노출되지 않도록 주의하십시오.)

NEXT_PUBLIC_API_URL=http://localhost:3000/api
ADMIN_TOKEN=your_secret_admin_token_123
NEXT_PUBLIC_ADMIN_PASSWORD=your_admin_password_here


3. 개발 서버 실행

npm run dev


4. 브라우저 접속

서버가 구동되면 http://localhost:3000으로 접속하여 결과를 확인합니다.

📁 프로젝트 구조 (Project Structure)

src/
├── app/
│   ├── page.tsx          # 메인 UI (도서 목록, 검색, 장바구니, 삭제 기능)
│   ├── globals.css       # 전역 스타일링
│   └── api/
│       ├── books/route.ts # 도서 추가/조회 (백엔드 로직 정상화 완료)

└── utils/
    ├── db.js             # SQLite 연동 및 공통 쿼리 유틸
    └── init-db.js        # 초기 데이터베이스 스키마 생성 스크립트
bookstore.db              # SQLite 로컬 데이터베이스 파일


✨ 상세 리팩토링 내역 (Improvement Report)

1. 배포 아키텍처 및 시크릿 보안 강화

현상 및 문제점: 관리자 비밀번호(admin123)가 코드에 평문으로 노출(Hardcoded Credentials)되어 있었으며, API 주소가 localhost로 고정되어 배포 시 통신이 불가능했습니다. 또한 서버리스 환경에서 SQLite 파일 DB가 휘발되는 구조적 한계가 존재했습니다.

조치 내용: 모든 민감 정보와 환경 종속적 데이터를 .env 환경변수로 격리하여 보안성을 확보하고 환경별 가변성을 높였습니다.

개선 계획: 데이터 영속성을 위해 AWS EC2/Docker 기반 컨테이너 배포와 외부 RDBMS(PostgreSQL 등) 마이그레이션을 제안합니다.

2. 인증/인가 시스템 및 보안 취약점 개선

현상 및 문제점:

인가 우회: URL 파라미터 조작(?admin=true)만으로 관리자 권한 탈취 가능.

SQL Injection: 사용자 입력값을 SQL문에 직접 연결하여 DB 전체가 탈취/삭제될 위험 발견.

조치 내용:

SQL 인젝션 방어: 모든 쿼리에 매개변수화된 쿼리(? 바인딩) 방식을 도입하여 공격을 원천 차단했습니다.

권한 로직 개선: 취약한 파라미터 방식을 폐기하고 sessionStorage 기반의 검증 체계로 전환했습니다.

개선 계획: bcrypt 암호화 및 JWT 기반의 서버 세션 검증 도입이 필요합니다.

3. 데이터 무결성 및 비즈니스 로직 정상화

현상 및 문제점:

백엔드 오류: 클라이언트 데이터를 무시하고 Math.random()으로 재고를 생성하는 로직 결함.

UI 버그: 자바스크립트 || 연산자의 특성상 재고 0이 '정보없음'으로 오표기되는 현상.

조치 내용:

로직 정상화: API가 클라이언트의 전송 데이터를 정확히 수신하여 DB에 적재하도록 수정했습니다.

널 병합 연산자 도입: ?? 연산자를 사용하여 재고 0이 화면에 정확히 표출되도록 개선했습니다.

4. TypeScript 타입 안정성 및 예외 처리

현상 및 문제점: 레거시 JS 마이그레이션 과정에서 unknown 타입 에러로 인해 빌드 및 런타임 불안정성 발생.

조치 내용: e as Error 명시적 타입 단언을 통해 예외 객체의 속성에 안전하게 접근하도록 수정하고 시스템 전반의 타입 추론을 강화했습니다.

5. 장바구니(Cart) UX/UI 고도화

현상 및 문제점: 상품 담기 기능만 존재하고 '삭제' 기능이 없어 사용자가 선택적으로 상품을 취소할 수 없는 치명적인 UX 결함 존재.

조치 내용: 장바구니 목록에 개별 삭제 버튼을 추가하고, filter 기반의 상태 업데이트 로직을 구현하여 즉각적인 사용자 피드백을 제공합니다.
